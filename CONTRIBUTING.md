# Contributing Guide

## Branch Naming Conventions

Use the standard structure: `feature/<issue-id>-summary` or `bugfix/<issue-id>-summary`.

## Commit Messages

Commits must follow the **Conventional Commits** specification:
```
type(scope): description
```

## Pull Request Guidelines

1. Every pull request requires green CI builds (typecheck, lint, test).
2. Peer review from at least one Code Owner is required before merge.
3. Keep changes scoped and self-contained.
